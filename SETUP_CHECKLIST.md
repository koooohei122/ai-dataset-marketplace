# サイト開設チェックリスト

サイトを開設するために必要な情報と手順をまとめました。

## 📋 必要な情報・アカウント

### 必須（サイト開設に必要）

- [ ] **GitHubアカウント**
  - 用途: コード管理、GitHub Pages（オプション）
  - 取得: https://github.com

- [ ] **Vercelアカウント**
  - 用途: サイトのホスティング（無料）
  - 取得: https://vercel.com（GitHubアカウントでログイン可能）

- [ ] **Supabaseアカウント**
  - 用途: データベース + ストレージ（無料）
  - 取得: https://supabase.com（GitHubアカウントでログイン可能）

### オプション（機能追加時に必要）

- [ ] **Google OAuth設定**
  - 用途: Googleアカウントでログイン
  - 取得: https://console.cloud.google.com
  - 必要な情報: Client ID, Client Secret

- [ ] **GitHub OAuth設定**
  - 用途: GitHubアカウントでログイン
  - 取得: GitHub Settings → Developer settings → OAuth Apps
  - 必要な情報: Client ID, Client Secret

- [ ] **Resendアカウント**
  - 用途: メール送信（無料: 3,000通/月）
  - 取得: https://resend.com

- [ ] **Stripeアカウント**（自動決済を使用する場合）
  - 用途: クレジットカード決済
  - 取得: https://stripe.com
  - 必要な情報: Secret Key, Publishable Key

- [ ] **ドメイン名**（オプション）
  - 用途: カスタムドメイン（例: aidataset-marketplace.com）
  - 取得: お名前.com、Google Domains等

## 🚀 サイト開設の手順

### Phase 1: プロジェクトセットアップ（私が準備します）

1. Next.jsプロジェクトの作成
2. 必要なパッケージのインストール
3. 基本的な構造の作成

### Phase 2: サービスアカウント設定（あなたが準備）

1. Supabaseプロジェクト作成
2. Vercelアカウント作成
3. OAuth設定（Google、GitHub）
4. その他のサービス設定

### Phase 3: 環境変数設定（一緒に設定）

1. 取得したAPIキーを環境変数に設定
2. データベース接続確認
3. 動作確認

### Phase 4: デプロイ（一緒にデプロイ）

1. Vercelにデプロイ
2. カスタムドメイン設定（オプション）
3. 動作確認
