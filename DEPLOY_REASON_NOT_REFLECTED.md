# ページが変わって見えない主な理由

## 結論
ローカルでは変更コミット済みでも、**本番にデプロイされていない** / **本番が別ブランチを参照している** と、URL上では変更は見えません。

## 典型パターン
1. GitHubにpushされていない
2. PRが未マージ
3. Vercelが `main` ではなく別ブランチを本番に設定している
4. デプロイ失敗（環境変数不足など）
5. ブラウザ/Edgeキャッシュで古いHTMLが表示される

## このリポジトリで特に起きやすい点
- ローカルブランチ `work` でコミットしているだけだと、本番URLには反映されない。
- `DATABASE_URL` 未設定だとビルドに失敗し、デプロイが止まる可能性がある。

## 反映確認の最短手順
1. 変更ブランチを `git push`
2. PRを作成して `main` にマージ
3. VercelのProduction Deployログを確認（成功ステータス）
4. 本番URLをハードリロード（Shift+Reload）
5. それでも変わらない場合、Vercelで最新デプロイを手動Redeploy
