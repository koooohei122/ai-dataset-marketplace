# AIデータ拡張機能 詳細仕様書

## 1. 概要

ユーザーがアップロードしたデータセットに対して、ローカルのCPU/GPUを使用してAIでデータ拡張を自動実行する機能。これにより、データセットのサイズが自然に増加し、より価値の高いデータセットとして販売できる。

## 2. 機能の目的

- **データセットの価値向上**: 元のデータに加えてAI生成データを追加することで、データセットのサイズと多様性が向上
- **ユーザーリソース活用**: ユーザーのローカルリソース（CPU/GPU）を使用することで、サーバーコストを削減
- **プライバシー保護**: データがローカルで処理されるため、機密データの保護が可能
- **自動化**: アップロード時にワンクリックでデータ拡張を実行

## 3. 対応データタイプ

### 3.1 画像データ
- **使用モデル**: Stable Diffusion、DALL-E、Midjourney API等
- **拡張方法**:
  - 画像バリエーション生成（元画像から類似画像を生成）
  - スタイル転送（異なるスタイルで再生成）
  - データオーグメンテーション（回転、拡大縮小、色調整等）
  - 条件付き生成（プロンプトベースの生成）
- **実装技術**:
  - ONNX Runtime for Web（ブラウザ上で実行）
  - Stable Diffusion WebUI API統合
  - WebGPU対応

### 3.2 テキストデータ
- **使用モデル**: GPT、BERT、T5等
- **拡張方法**:
  - パラフレーズ生成（同じ意味の異なる表現）
  - 要約・拡張（短いテキストを詳細化、または逆）
  - 翻訳・逆翻訳（多言語データセットの生成）
  - スタイル変換（フォーマル↔カジュアル等）
- **実装技術**:
  - Transformers.js（ブラウザ上でLLM実行）
  - Ollama統合（ローカルLLM）
  - Web Workers（バックグラウンド処理）

### 3.3 音声データ
- **使用モデル**: TTS（Text-to-Speech）、Voice Cloning等
- **拡張方法**:
  - 音声バリエーション生成（異なる話者、感情）
  - ノイズ追加（現実的な環境音の追加）
  - ピッチ・速度変更
  - 背景音楽の追加・除去
- **実装技術**:
  - Web Audio API
  - 音声生成モデル（Coqui TTS等）

### 3.4 表形式データ（CSV、JSON等）
- **使用モデル**: GAN、VAE、Tabular GAN等
- **拡張方法**:
  - 合成行データ生成（統計的特性を保持）
  - 欠損値補完
  - 外れ値生成
  - カテゴリカルデータの拡張
- **実装技術**:
  - CTGAN、TVAE等の表形式生成モデル
  - WebAssembly（高速計算）

### 3.5 動画データ
- **使用モデル**: 動画生成モデル（将来的に）
- **拡張方法**:
  - フレーム補間
  - 動画スタイル転送
  - シーン拡張

## 5. 技術実装

### 5.1 クライアントサイド処理（推奨）

#### 5.1.1 ブラウザ上でのAI実行
- **WebGPU**: GPUアクセラレーション
- **WebAssembly**: 高速計算
- **Web Workers**: バックグラウンド処理
- **IndexedDB**: モデルとデータのキャッシング

#### 5.1.2 モデルのロード
```typescript
// 画像拡張の例
import { loadStableDiffusion } from '@/lib/augmentation/image';

const model = await loadStableDiffusion({
  quality: 'high',
  useGPU: true,
  cache: true // IndexedDBにキャッシュ
});

// 画像生成
const variants = await model.generateVariations(
  originalImage,
  count: 2, // 2つのバリエーションを生成
  options: {
    strength: 0.7, // 元画像との類似度
    guidance: 7.5
  }
);
```

#### 5.1.3 進捗管理
```typescript
// WebSocketまたはServer-Sent Eventsで進捗を送信
const progressCallback = (progress: AugmentationProgress) => {
  updateUI({
    progress: progress.percentage,
    currentFile: progress.currentFile,
    completed: progress.completed,
    total: progress.total,
    estimatedTime: progress.estimatedTimeRemaining
  });
};
```

### 5.2 サーバーサイド処理（フォールバック）

クライアント側で処理できない場合の代替手段：

- **条件**:
  - GPUが利用できない
  - ブラウザがWebGPUに対応していない
  - データが非常に大きい
  - ユーザーがサーバー処理を選択

- **実装**:
  - サーバー側でAI処理を実行
  - キューシステム（Bull/BullMQ）でジョブ管理
  - 処理完了をWebhookまたはポーリングで通知

### 5.3 データ管理

#### 5.3.1 ファイルの識別

## 6. パフォーマンス最適化

### 6.1 モデルの最適化
- **量子化**: INT8量子化でモデルサイズと速度を改善
- **プルーニング**: 不要なニューロンを削除
- **蒸留**: より小さなモデルに知識を転移

### 6.2 バッチ処理
- 複数ファイルをまとめて処理
- GPUの並列処理能力を最大限活用

### 6.3 キャッシング
- モデルをIndexedDBにキャッシュ（初回ロード後は高速）
- 同じファイルの再生成を回避

### 6.4 プログレッシブ生成
- 低解像度→高解像度の段階的生成
- ユーザーが早期に結果を確認可能
