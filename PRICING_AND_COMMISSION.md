# 価格設定・手数料システム 詳細仕様

## 1. 概要

AIデータセットマーケットプレイスでは、Epic Games Fabのような仕組みを採用し、**無料データセットと有料データセットの両方**をサポートします。出品ユーザーは自由に価格を設定でき、プラットフォーム運営者は販売額から一定の手数料（マージン）を取得します。

## 2. 価格設定

### 2.1 出品ユーザーの価格設定

- **価格設定の自由度**
  - 0円以上で自由に価格を設定可能
  - 0円 = 無料データセット
  - 有料データセットも自由に価格設定可能（例: ¥100、¥1,000、¥50,000等）

- **価格変更**
  - いつでも価格を変更可能
  - 既に購入済みのユーザーには影響なし（購入時の価格が適用）
  - 価格変更履歴を記録（オプション）

- **価格制限（オプション）**
  - 運営者が最小価格・最大価格を設定可能
  - 例: 最小¥100以上、最大¥1,000,000以下

### 2.2 無料データセット

- **特徴**
  - Epic Games Fabのような仕組み
  - 購入ボタンが「無料でダウンロード」に変更
  - 購入フローが簡略化（決済不要、即座にダウンロード可能）
  - レビュー・評価は有料と同様に可能

- **用途**
  - プロモーション目的
  - コミュニティへの貢献
  - サンプルデータの提供
  - オープンソースデータセットの配布

- **手数料**
  - 無料データセットには手数料なし（¥0のため）

## 3. プラットフォーム手数料（マージン）

### 3.1 手数料の仕組み

```
販売価格 = ¥10,000
プラットフォーム手数料率 = 15%
決済手数料 = 3.6% + 40円（Stripeの場合）

計算:
- プラットフォーム手数料 = ¥10,000 × 15% = ¥1,500
- 決済手数料 = ¥10,000 × 3.6% + 40円 = ¥400
- 販売者への支払い = ¥10,000 - ¥1,500 - ¥400 = ¥8,100
```

### 3.2 手数料率の設定

- **デフォルト手数料率**: 15%
  - 業界標準（Steam: 30%, Epic Games: 12%, Itch.io: 10-30%）
  - バランスの取れた設定

- **可変手数料率**
  - 運営者が設定可能（例: 10-20%）
  - カテゴリごとに異なる手数料率を設定可能（オプション）
  - 例: 画像データセット: 15%、テキストデータセット: 12%

- **手数料率の変更**
  - 既存の購入には影響なし
  - 新しい購入から新しい手数料率が適用

### 3.3 決済手数料の扱い

- **オプション1: 販売者負担（推奨）**
  ```
  販売者への支払い = 販売価格 - プラットフォーム手数料 - 決済手数料
  ```
  - 販売者が決済手数料も負担
  - 運営者の収益が明確

- **オプション2: 運営者負担**
  ```
  販売者への支払い = 販売価格 - プラットフォーム手数料
  運営者の実収益 = プラットフォーム手数料 - 決済手数料
  ```
  - 運営者が決済手数料を負担
  - 販売者への支払いが増える

- **オプション3: 購入者負担**
  ```
  購入価格 = 販売価格 + 決済手数料
  販売者への支払い = 販売価格 - プラットフォーム手数料
  ```
  - 購入者が決済手数料を負担
  - 販売価格がそのまま販売者への支払い基準

### 3.4 手数料計算の例

#### 例1: 有料データセット（¥10,000、手数料15%、決済手数料販売者負担）

```
販売価格: ¥10,000
プラットフォーム手数料: ¥10,000 × 15% = ¥1,500
決済手数料（Stripe）: ¥10,000 × 3.6% + 40円 = ¥400
販売者への支払い: ¥10,000 - ¥1,500 - ¥400 = ¥8,100
運営者の収益: ¥1,500
```

#### 例2: 無料データセット

```
販売価格: ¥0
プラットフォーム手数料: ¥0
決済手数料: ¥0
販売者への支払い: ¥0
運営者の収益: ¥0
```

#### 例3: 高額データセット（¥100,000、手数料15%）

```
販売価格: ¥100,000
プラットフォーム手数料: ¥100,000 × 15% = ¥15,000
決済手数料（Stripe）: ¥100,000 × 3.6% + 40円 = ¥3,640
販売者への支払い: ¥100,000 - ¥15,000 - ¥3,640 = ¥81,360
運営者の収益: ¥15,000
```

## 4. 売上管理

### 4.1 販売者ダッシュボード

販売者は以下の情報を確認できます：

- **総売上**: 全データセットの合計販売額
- **プラットフォーム手数料**: 合計手数料額
- **決済手数料**: 合計決済手数料額
- **受取額**: 販売者への支払い額
- **未払い額**: まだ支払われていない金額
- **支払い履歴**: 過去の支払い記録

### 4.2 運営者ダッシュボード

運営者は以下の情報を確認できます：

- **総売上**: 全データセットの合計販売額
- **プラットフォーム手数料収益**: 合計手数料収益
- **決済手数料**: 合計決済手数料（運営者負担の場合）
- **純利益**: プラットフォーム手数料 - 決済手数料
- **データセット別の売上**: 各データセットの売上と手数料
- **販売者別の売上**: 各販売者の売上と手数料

### 4.3 支払いスケジュール

- **手動支払い（無料運営向け）**
  - 販売者が申請
  - 運営者が確認・承認
  - 銀行振込またはPayPalで支払い
  - 月次または都度

- **自動支払い（有料サービス使用時）**
  - 自動的に販売者アカウントに残高を追加
  - 一定額（例: ¥10,000）以上で自動出金
  - または月次自動出金

## 7. 最適なマージン割合の推奨

### 7.1 業界標準との比較

| プラットフォーム | 手数料率 |
|----------------|---------|
| Steam | 30% |
| Epic Games Store | 12% |
| Itch.io | 10-30% |
| Apple App Store | 30% (15% for small businesses) |
| Google Play Store | 30% (15% for first $1M) |
| **推奨（本プラットフォーム）** | **15%** |

### 7.2 15%を推奨する理由

1. **バランス**: 運営者と販売者の両方にとって公平
2. **競争力**: Epic Games Storeと同等で競争力がある
3. **持続可能性**: プラットフォーム運営に必要な収益を確保
4. **成長**: 手数料収益で機能拡張・マーケティングが可能

### 7.3 手数料率の調整

- **初期段階（低手数料）**: 10-12%で販売者を集める
- **成長段階（標準手数料）**: 15%で安定運営
- **成熟段階（カテゴリ別）**: カテゴリごとに最適化（10-20%）

## 8. 実装のポイント

### 8.1 手数料計算の実装

```typescript
function calculateFees(
  price: number,
  platformFeeRate: number,
  paymentFeeRate?: number,
  paymentFeeFixed?: number
) {
  const platformFee = price * (platformFeeRate / 100);
  const paymentFee = paymentFeeRate 
    ? price * (paymentFeeRate / 100) + (paymentFeeFixed || 0)
    : 0;
  const sellerAmount = price - platformFee - paymentFee;
  
  return {
    price,
    platformFee,
    platformFeeRate,
    paymentFee,
    sellerAmount
  };
}
```

## 9. まとめ

- **無料データセット**: Epic Games Fabのような仕組みで、無料でもダウンロード可能
- **有料データセット**: 出品ユーザーが自由に価格設定可能
- **プラットフォーム手数料**: デフォルト15%（業界標準、調整可能）
- **透明性**: 販売者と運営者の両方が手数料を明確に確認可能
- **柔軟性**: 手数料率、決済手数料の負担者を設定可能

この仕組みにより、持続可能なプラットフォーム運営と、販売者への公平な報酬の両立が可能になります。
